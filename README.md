# JJ Accounting Cloud

Read-only Next.js dashboard for the JJ Accounting desktop app.

## Durable Storage

The deployed site stores the latest desktop SQLite snapshot in Vercel Blob. This avoids losing data when Vercel serverless functions restart or move between instances.

Required production environment variable:

```text
BLOB_READ_WRITE_TOKEN
```

Optional environment variables:

```text
JJ_SYNC_TOKEN
JJ_BLOB_DB_PATH=jj-accounting/jj-accounting.sqlite
JJ_BLOB_META_PATH=jj-accounting/sync-meta.json
```

## Vercel Setup

1. Create or connect a Vercel Blob store for this project.
2. Add `BLOB_READ_WRITE_TOKEN` to the Vercel project environment variables.
3. Redeploy the website.
4. Open the website, generate an access key, and paste that key into the desktop app Cloud Sync settings.
5. Run Sync Now from the desktop app.

For local development without a Blob token, the app falls back to `website-data/`. That folder is ignored because it can contain private accounting data.

## Troubleshooting Blob Sync

If the website says the database exists but the access key metadata is missing, the SQLite file and `sync-meta.json` are not both present at the configured Blob paths. Generate a fresh website access key, paste that key into the desktop app, and run Sync Now so the app uploads both the database and metadata to Blob.

If the desktop app says the cloud sync password is invalid after moving to Blob, the app is still using an old access key from local/serverless storage or another Blob path. Use the latest key generated by the deployed website, or verify `JJ_BLOB_DB_PATH` and `JJ_BLOB_META_PATH` point to the same store and folder used by the deployed site.
